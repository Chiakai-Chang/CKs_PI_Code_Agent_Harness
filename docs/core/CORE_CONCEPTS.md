# 🧪 CK's Pi Code Agent Harness：核心設計理念與加強原理

這份文件詳細說明了本專案如何在原生的 `Pi Coding Agent` 基礎上，透過整合輔助擴充與專家規則，以期提升開發效率。本專案目前定位為實驗性質，旨在探索更佳的 AI 開發輔助路徑，並不保證完全優於原生環境。

---

## 1. 攔截審查擴充 (ECC Hooks Bridge)
**「在執行指令或修改檔案前，進行安全檢查與格式驗證。」**

*   **它的角色**：它是 Pi 助手的工具執行防火牆。
*   **運作原理**：利用 Pi 的擴充介面，我們在 AI 「執行指令 (Bash)」或「編輯檔案 (Edit)」的前後插入了擴充程序。它會嘗試呼叫 `Everything Claude Code (ECC)` 的檢查規則（若本地環境存在）進行「安全與語法審核」。
*   **整合目標**：
    *   **品質門檻 (Quality Gate)**：在存檔前驗證括號完整性與排版。
    *   **安全防護**：防範 AI 意外修改或刪除金鑰（`.env`）或在關鍵分支執行高風險指令。
*   **跨平台優化**：
    *   由於這類檢查通常依賴絕對路徑，我們在 `setup.py` 中實現了自動偵測與路徑注入，以避免手動配置造成的路徑解析失效。

## 2. 工程紀律引導 (Superpowers - Native)
**「引導 AI 在選擇工具前先進行結構化的思考與規劃步驟。」**

*   **它的角色**：它是 Pi 助手的元指令（Meta-instructions）引導規程。
*   **運作原理**：本專案透過 **Native Mapping** 整合了 `obra/superpowers` 的提示詞設計。強制 Pi 在每一輪對話開始前，先反思任務類型、是否需要編寫測試、或是否需要建立實體計畫。
*   **整合目標**：引導 AI 減少盲目的工具調用，優先採用 `/plan`（規劃）與 `/tdd`（測試驱动開發）等結構化流程，以提升開發過程的嚴謹性與可追蹤性。
*   **整合優勢**：藉由 Submodule 原生映射，能同步接收上游最新的工程紀律更新，而不會遺失 Reference 目錄下的參考資源。

## 3. 代碼結構解讀 (Understand-Anything)
**「協助 AI 分析與建構代碼結構關係圖，緩解架構迷航。」**

*   **它的角色**：它是專案架構分析與 Onboarding 的輔助工具。
*   **運作原理**：對接 `Understand-Anything` 工具。透過 `/understand` 指令引導 AI 掃描專案並建立代碼邏輯關係圖，將實體與關係以結構化方式保存。
*   **整合目標**：
    *   **架構解讀 (Onboarding)**：利用 `/understand-onboard`，協助 AI 在新進專案時快速梳理模組層級與核心入口。
    *   **影響範圍評估**：在重構時提供更精確的相依性參考，避免盲目修改代碼。
*   **優化細節**：引進過濾清單與增量掃描概念，以減少大型專案中無謂的 Token 與時間消耗。

## 4. 專案知識沉澱 (LLM Wiki - Karpathy Pattern)
**「實驗性地沉澱專案研究日誌與架構文件，形成知識庫。」**

*   **它的角色**：它是專案的長期知識沉澱載體。
*   **運作原理**：嘗試實作 Andrej Karpathy 提出的 LLM Wiki 概念。當 AI 獲得新的架構發現或開發日誌時，將其整理為具備雙向鏈接的 Markdown 文件，隨專案版本控制保存。
*   **整合目標**：解決 AI 會話結束後知識即消失的問題。透過文件沉澱，使後續的 AI 代理人或新開發者能快速檢索專案的設計脈絡。
*   **維護防禦**：配合 Link Checker，防止雙向鏈接隨檔案重構而失效。

## 5. 實體計畫板 (Manus-style Planning / Planning with Files)
**「利用本地檔案保存任務規劃與執行狀態，緩解 Context 忘記問題。」**

*   **它的角色**：它是跨會話的計畫與記憶快照。
*   **運作原理**：原生 AI 的記憶完全依賴對話上下文，當對話變長時極易產生記憶漂移或遺忘。我們透過擴充引導 AI 將任務分解寫入專案目錄下的 `task_plan.md` 等實體檔案中。
*   **整合目標**：
    *   **跨會話接關**：重啟 `pi` 後，AI 能透過讀取本地計畫檔案快速還原執行進度，無需重新詢問。
    *   **進度可視化**：將抽象的推理步驟實體化，便於人類開發者審查與介入。

## 6. 本地環境規格感應 (Smart & Hardware Awareness)
**「在環境部署時自動感應本地規格，推薦合適的模型設定。」**

*   **它的角色**：它是部署時的硬體適應層。
*   **運作原理**：在執行 `setup.py` 時，腳本會自動調用系統探針（如 `wmic` 或 `nvidia-smi`）與本地 LLM API（如 Ollama 的型號與規格接口），抓取實際硬體資源。
*   **整合目標**：自動推薦適合該主機體質的 Context Window 與 Max Tokens，降低手動填寫參數出錯而導致顯存溢出 (OOM) 或運算卡死的機率。

## 7. 一鍵環境自癒 (Zero-Friction Installer)
**「簡化環境建置與 Git 權限、編碼等常見障礙的排除流程。」**

*   **它的角色**：專案初始化引導腳本。
*   **運作原理**：將權限提權、編碼設定（Windows UTF-8 強制開啟）、Git 信任目錄修正（Safe Directory）、模型配置、以及 Submodule 還原封裝至 `install.bat` / `install.sh`。
*   **整合目標**：降低跨平台（Windows/macOS/Linux）手動建置環境的摩擦力，確保複製本專案後即可快速進入開發狀態。

## 8. 行為誠信驗證 (AIxBDD - Integrity First)
**「實踐測試優先（BDD）的開發閉環，確保代碼有據可循。」**

*   **它的角色**：開發品質與誠信門檻。
*   **運作原理**：引入 `Waterball AIxBDD` 概念，引導 AI 實施 `Red -> Green -> Refactor` 的開發節奏。
*   **整合目標**：要求 AI 必須先撰寫出失敗的測試（證明其確實理解需求），方可編寫實作代碼，並在需求異動時引導進行測試與規格的調和（Reconciliation），從源頭控制代碼退化。

## 9. 實驗性多代理協作 (OMC Multi-Agent Factory)
**「實驗性地對接 OMC 多代理協作流程，分解任務角色。」**

*   **它的角色**：多代理協作模式對接器。
*   **運作原理**：引入 `oh-my-claudecode (OMC)` 的代理人角色定義（如 Architect, Executor, Critic），並在規劃時嘗試分配不同角色的職責。
*   **整合目標**：探索當單一 Agent 面對複雜系統架構力有未逮時，如何透過多角色分工與 Sisyphus 重試引擎，提高代碼生成與故障修復的成功機率。

## 10. 環境治理與 C.A.S.E. 框架對接
**「對接 C.A.S.E. 設計，提供開發環境的標準對齊。」**

*   **它的角色**：專案環境治理的理念對齊。
*   **設計理念**：本專案引入了 [C.A.S.E. 框架](https://github.com/Chiakai-Chang/Local-Agent-Workspace/tree/main/C.A.S.E._Framework)（Context, Action, State, Environment）治理思維。
*   **整合目標**：在 `pi-extensions` 內部的橋接擴充中，將 C.A.S.E. 思想作為環境與狀態的對齊依據，確保 AI 在執行時其 Action (工具調用)、Context (上下文傳遞) 與 State (計畫狀態) 能在該 Environment 中被清晰追蹤與校準，避免行為脫軌。

### 功能模組整合總覽表

| 模組名稱 | 白話功能 | 整合點 | 解決的實際問題 |
| :--- | :--- | :--- | :--- |
| **ECC Bridge** | 攔截審查機制 | **安全與語法檢查** | 防止 AI 寫出殘缺代碼或執行危險指令。 |
| **Superpowers** | 元指令引導 | **工程規劃紀律** | 強制 AI 在實作前進行 TDD / 規劃反思。 |
| **Understand** | 代碼 GPS | **架構依存解讀** | 解決接手大型/舊專案時結構解讀費時的痛點。 |
| **LLM Wiki** | 專案知識庫 | **雙向鏈接文件** | 避免開發脈絡隨對話會話結束而丟失。 |
| **Manus Planning** | 實體計畫板 | **檔案快照記憶** | 緩解長對話導致 AI 遺忘任務目標的缺陷。 |
| **AIxBDD** | 行為誠信 | **BDD 測試閉環** | 確保代碼實現皆有對應的行為測試依據。 |
| **OMC Teams** | 多代理協作 | **角色任務分工** | 探索複雜任務的多視角審查與自動修復。 |
| **C.A.S.E. Align** | 環境治理 | **四維狀態治理** | 確保 AI 行為與當前開發環境高度對齊。 |
| **Smart Detect** | 本地規格偵測 | **參數自適應推薦** | 避免手動設定不當引發显存溢出或卡死。 |
| **Healing Installer** | 一鍵自癒部署 | **多平台建置簡化** | 快速排除 Windows 權限、Git 信任等常見阻礙。 |
