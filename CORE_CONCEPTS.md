# 🧠 CK's Pi Code Agent Harness：核心設計理念與加強原理

這份文件詳細說明了本專案如何在原生的 `Pi Coding Agent` 之上，透過五大核心系統將其改造成「資深工程師等級」的開發環境。

---

## 1. 自動糾察隊系統 (ECC Hooks Bridge)
**「在 AI 闖禍之前，先由專家攔截。」**

*   **它的角色**：它是 Pi 助手的「良知」與「規範執行者」。
*   **運作原理**：利用 Pi 的擴充介面，我們在 AI 「打算執行指令 (Bash)」或「按下存檔 (Edit)」的**瞬間**插入了檢查程序。它會呼叫 `Everything Claude Code (ECC)` 的高品質檢查腳本進行「秒級審查」。
*   **加強了什麼**：
    *   **品質門檻 (Quality Gate)**：如果 AI 寫的代碼少了一個括號或是不符合規範，糾察隊會直接擋下並叫 AI 重寫。
    *   **安全防護**：防止 AI 意外刪除 `.env` 檔案、把金鑰推送到 Git 或是在 `main` 分支進行危險操作。
*   **潛在問題與處理**：
    *   **問題**：這類腳本通常依賴「絕對路徑」，換電腦或換資料夾就失效。
    *   **處理**：我們實作了「絕對路徑自動注入」。在安裝時，系統會自動定位本倉庫位置並寫入 Pi 的全域設定，確保糾察隊不管在哪裡都能找到「家」。

## 2. 專家直覺系統 (Superpowers)
**「不再只是直覺反應，而是先思考、再選工具。」**

*   **它的角色**：它是 Pi 助手的「職業訓練」與「作業標準書 (SOP)」。
*   **運作原理**：透過一套「元指令 (Meta-instructions)」，強制 Pi 在每一輪對話開始前，先反思：*「這題是 UI 嗎？這題要寫測試嗎？」* 然後主動去技能庫 (Skills) 找最適合的工具。
*   **加強了什麼**：讓 AI 從「會寫代碼的機器」進化成「懂開發流程的專家」。它會主動發起 `/tdd` (測試驅動開發) 或 `/plan` (詳細規劃)，而不是沒頭沒腦地直接改代碼。
*   **潛在問題與處理**：
    *   **問題**：AI 有時會變得太過「話多」或過度思考。
    *   **處理**：我們精選了最核心的技能包，剔除重複與不穩定的實驗性技能，並在 Rules 中定義了優先級，維持高效率回應。

## 3. 戰術筆記系統 (Manus-style Planning)
**「給 AI 一塊永遠不會斷電、不會失憶的黑板。」**

*   **它的角色**：它是 Pi 助手的「長期記憶體」。
*   **運作原理**：原生 AI 的記憶存在「對話紀錄」裡，講太長就會忘記前面。我們強迫 Pi 將計畫實體化為專案中的 `task_plan.md`、`findings.md` 等檔案。
*   **加強了什麼**：
    *   **跨對話延續性**：你今天做一半，關掉電腦，明天再開，Pi 讀一下檔案就能立刻「接關」，完全不會問你「我們剛才聊到哪？」。
    *   **結構化追蹤**：計畫變得可視、可管理。
*   **潛在問題與處理**：
    *   **問題**：專案資料夾會多出一些臨時檔案，顯得雜亂。
    *   **處理**：我們建立了自動化橋接，任務完成後 Pi 會主動詢問是否要「歸檔」或「清理」這些戰術檔案。

## 4. 硬體透視系統 (Smart & Hardware Awareness)
**「讓 AI 看清楚自己跑在什麼等級的引擎上。」**

*   **它的角色**：它是 Pi 助手的「儀表板」。
*   **運作原理**：安裝時，我們透過 API 探針（深入探測 `llama.cpp` 的 `n_ctx` 欄位）與系統指令（`nvidia-smi`、`wmic`）獲取「真值」。
*   **加強了什麼**：
    *   **極限性能調優**：自動偵測並設定 192k (196,608) 這種超長 Context，讓 AI 能一次讀完你的整個大專案。
    *   **穩定性防禦**：如果偵測到顯存 (VRAM) 只有 12GB 卻要跑大型模型，系統會警告並自動下修推薦值，防止電腦當機。
*   **潛在問題與處理**：
    *   **問題**：API 路徑千奇百怪（有的加 `/v1`，有的在根目錄）。
    *   **處理**：我們寫了「全能探針」，會自動切換 `127.0.0.1`、剝離路徑、並解析嵌套 JSON，確保 99% 的本地模型都能被精準識別。

## 5. 自癒式安裝系統 (The Automated Workshop)
**「讓複雜的配置變為『按一下就好』。」**

*   **它的角色**：它是進入這個強大環境的「無障礙坡道」。
*   **運作原理**：將權限檢查、編碼修正、Git 信任、環境補全全部封裝在一個 `install.bat` / `install.sh` 內。
*   **加強了什麼**：
    *   **自動提權**：權限不足？它會自動問你要不要用管理員重新執行。
    *   **Git 信任自動修復**：遇到 Windows 磁碟權限問題 (Dubious Ownership)？它會自動執行 `safe.directory` 修正。
    *   **零亂碼體驗**：強制開啟 UTF-8 支援，確保中文環境不再出現亂碼。
*   **處理邏輯**：採用的「Enter-Centric」設計，專家可以微調，新手可以一路 Enter 到底，實現真正的專業級一鍵部署。
